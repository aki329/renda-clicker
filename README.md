# 連打ツール

普通のWebサイト上で、指定したCPSのクリックを開始/停止します。上限は100CPSです。

## 拡張機能を入れない使い方

1. ローカルサーバーを起動する。
2. `http://localhost:8000/bookmarklet.html` を開く。
3. 「連打ツール」ボタンをブックマークバーへドラッグする。
4. 使いたいWebサイトを開く。
5. 保存した「連打ツール」ブックマークを押す。
6. 右下のパネルでCPSと開始/停止キーを変える。

ドラッグできない時は、ページ下のURL文字列をコピーして、手動でブックマークのURL欄へ入れてください。
古いブックマークには変更が反映されないため、更新後はブックマークを作り直してください。

## 学校Chromebookの場合

学校の制限を解除・回避する使い方はしません。学校で `javascript:` ブックマークが許可されている場合だけ使えます。

Chromebookでは、Macの `file:///Users/aki/.../chromebook.html` は開けません。ダウンロードなしで使う場合は、GitHub Pagesなどで公開したURLを開きます。

公開URLの形:

```text
https://<GitHubユーザー名>.github.io/<リポジトリ名>/chromebook.html
```

1. 公開URLの `chromebook.html` を開く。
2. テスト欄の「コピー」を押して、ブックマークのURL欄に貼る。
3. そのブックマークを押して「連打ツールのテストOK」が出るか確認する。
4. OKが出たら連打コード欄の「コピー」を使って、連打ツール用ブックマークを作る。
5. 通常コードが長すぎて動かない場合だけ、公開URLで開いた `chromebook.html` から「短いコード」を使う。

コピーできない時は「全部選択」を押して手動でコピーしてください。保存できない、または `javascript:` が消える場合は、学校の管理設定で禁止されています。その場合は先生か学校管理者に確認してください。

`file://` で `chromebook.html` を開いた場合も、通常コードは作れます。短いコードは公開URLで開いた時だけ使えます。

## GitHub Pagesで公開する手順

このフォルダは静的サイトとしてそのまま公開できます。外部へ公開する作業なので、実際に公開する前に内容を確認してください。

1. GitHubで新しいリポジトリを作る。
2. このフォルダのファイルをリポジトリへ入れる。
3. GitHubの `Settings` → `Pages` を開く。
4. `Deploy from a branch` を選ぶ。
5. `main` ブランチの `/root` を選んで保存する。
6. 表示されたURLの最後に `/chromebook.html` を付けて開く。

専用アプリとしてダウンロードする方式は、学校Chromebookでは管理者に止められることが多く、Webアプリでは他サイト上をクリックできません。連打ツールとしてはブックマーク版、または管理者が許可したChrome拡張版を使います。

管理者向けの正規ルート:

- URLの許可/ブロック設定: https://support.google.com/chrome/a/answer/7532419
- 拡張機能の自動インストール: https://support.google.com/chrome/a/answer/6306504

## Chrome拡張として使う方法

1. Chromeで `chrome://extensions` を開く。
2. 右上の「デベロッパー モード」を有効にする。
3. 「パッケージ化されていない拡張機能を読み込む」を押す。
4. このフォルダを選ぶ。
5. 使いたいWebサイトを開く。すでに開いていたページは一度再読み込みする。
6. クリックしたい場所にマウスを置く。
7. 初期設定では `Option+Shift+C` で開始/停止する。

Edgeの場合は `edge://extensions` から同じ手順で読み込めます。

## 設定

- 初期値は `30 CPS` です。
- `1` から `100` CPSの範囲で変更できます。
- `10`、`30`、`60`、`100` のプリセットボタンがあります。
- 「開始/停止キー」の「変更」を押すと、開始/停止キーを変えられます。
- `Esc` は常に緊急停止です。
- 右下の表示は、ブックマークレット版では `×` で非表示、表示ボタンで再表示できます。
- ブックマークレット版では「このサイトから完全削除」で、現在のページからツール本体と保存設定を消せます。
- 拡張機能版では、拡張アイコン内の「表示」で切り替えできます。
- 表示位置は左下/右下で切り替えできます。
- Cookie Clickerの大きいクッキー上では、通常クリックが通りにくい場合の専用処理を使います。

## テスト

すでにローカルサーバーが動いている場合は、次を開くだけです。

```text
http://localhost:8000/
```

ローカルで確認するときは、このフォルダで次を実行します。

```bash
python3 -m http.server 8000
```

そのあと、ブラウザで次を開きます。

```text
http://localhost:8000/tests/cps-test.html
```

テストボタンにマウスを置いて `Option+Shift+C` を押すと、直近1秒のCPSを確認できます。

## 注意

- 初期状態は必ず停止です。
- `chrome://`、`edge://`、拡張機能ページ、ブラウザ外アプリでは動きません。
- 支払い、削除、投稿、送信、設定変更の画面では開始しないでください。
- この拡張はブラウザ内でクリックイベントを送ります。OS全体の本物のマウスクリックではありません。
- サイトによっては、拡張から送ったクリックを受け付けない場合があります。
